# تقرير الحالة الحالية لمشروع Vertex Automation

## نظرة عامة

المشروع عبارة عن نظام لأتمتة معالجة طلبات الرفع المساحي الخاصة بشركة Vertex.

البرنامج يستقبل فولدرات المشروع كما يتم تسليمها من الشركة، ثم يقوم تدريجيًا بـ:

1. قراءة بيانات الطلبات.
2. ربطها بملف Excel.
3. تحديث ملفات الـ Shapefile.
4. استخراج بيانات بطاقة الرقم القومي (قيد التطوير).
5. إنشاء فولدر نتائج جاهز للتسليم.

---

# الهيكل العام

المشروع مقسم إلى جزئين رئيسيين:

```
app/
```

وهو الجزء الخاص بالـ Workflow وإدارة المشروع.

و

```
ai/
```

وهو الجزء الخاص بالرؤية الحاسوبية والذكاء الاصطناعي.

---

# أولاً: جزء app

هذا الجزء يعتبر مستقرًا إلى حد كبير.

## 1- Scanner

يقوم بمسح فولدر Vertex بالكامل.

يكتشف تلقائيًا:

* ملفات Excel
* مجلدات التواريخ
* ملفات SHP
* مجلدات الطلبات
* مجلد الصور

ويحولها إلى Objects داخل البرنامج.

---

## 2- Excel Reader

يقوم بقراءة ملف Excel بالكامل باستخدام Pandas.

ثم تحويل كل صف إلى:

```
ExcelRequest
```

---

## 3- Linker

يقوم بربط كل طلب داخل الفولدر مع السجل الخاص به داخل Excel باستخدام:

```
surv_num
```

بعدها يصبح لكل Request:

```
request.excel
```

ويحتوي على:

* اسم المواطن
* رقم الطلب
* المحافظة
* الشركة
* نوع الطلب
* ...
  إلخ

---

## 4- GIS Update

يقرأ ملف الـ SHP.

ثم يحدث الحقول التالية:

* الاسم
* رقم الطلب
* المحافظة
* القسم
* الموقع
* النوع

ثم يحفظ:

```
*_updated.shp
```

---

## 5- Export Result

تم تنفيذه بالكامل تقريبًا.

يقوم بـ:

لكل Feature داخل الـ Updated SHP:

١. إنشاء فولدر باسم:

```
Survey Number
```

٢. تصدير Feature واحد فقط داخل SHP مستقل.

٣. تسمية الملف تلقائيًا.

مثال:

```
build.shp

land.shp

boundary.shp
```

حسب:

```
النوع

والوصف
```

٤. نسخ جميع صور الطلب إلى نفس الفولدر.

الناتج النهائي:

```
Result/

    7-6-2026/

        A-B28....

            الصور

            shp/

                build.shp
```

# ثانياً: جزء AI

هذا الجزء ما زال تحت التطوير.

لكن تم الانتهاء من جزء كبير من الـ Pipeline.

---

# 1- Card Detection

تم الانتهاء.

الموديل المستخدم:

Grounding DINO

يقوم بالبحث داخل صورة الموقع عن:

```
identity card
```

ثم يرجع Bounding Box للبطاقة.

بعدها يتم قص البطاقة.

---

# 2- Card Cropper

تم الانتهاء.

يقوم بقص البطاقة فقط من الصورة.

ويحفظها داخل:

```
output/cards/card.jpg
```

---

# 3- Image Preprocessing

تم الانتهاء.

يطبق:

* Resize ×2
* CLAHE
* Bilateral Filter
* Grayscale

ثم يحفظ:

```
card_processed.jpg
```

---

# 4- Region Extraction

تم الانتهاء.

بدلاً من إرسال البطاقة كاملة إلى OCR.

يتم استخراج منطقتين فقط.

## Name Region

يتم قص الجزء الخاص باسم المواطن.

## ID Region

يتم قص الجزء الخاص بالرقم القومي.

مع إضافة Padding لتحسين الدقة.

الناتج:

```
name.jpg

id.jpg
```

---

# 5- OCR

الحالة الحالية:

قيد التطوير.

تمت تجربة عدة حلول.

## PaddleOCR

المميزات

* جيد مع الإنجليزية.

العيوب

* ضعيف مع الرقم القومي العربي.
* يحتاج Detection لكل صورة.
* دقة غير كافية.

---

## Surya OCR

تمت تجربته.

المميزات

* أفضل مع العربية.
* أسرع.
* Recognition قوي.

المشكلة الحالية

يوجد اختلاف في API بين الإصدار الموجود والإصدار المستخدم في التوثيق.

مما أدى إلى توقف مرحلة الـ Recognition.

تم تحديد سبب المشكلة، لكن لم يتم الانتهاء من دمج Surya داخل الـ Pipeline.

---

# 6- Matching

الملف موجود.

لكن لم يبدأ العمل عليه.

الفكرة ستكون:

OCR

↓

استخراج الاسم

↓

مطابقته مع اسم المواطن الموجود في Excel

ثم:

استخراج الرقم القومي.

---

# لماذا سيتم عمل Matching؟

لأن الرقم القومي قد يُقرأ بخطأ رقم أو رقمين.

وجود الاسم داخل Excel يسمح بالتحقق من أن البطاقة تخص نفس المواطن قبل تسجيل الرقم القومي.

---

# 7- Extractor

الملف موجود.

ولم يبدأ تنفيذه.

سيكون مسئولًا عن:

```
OCR Text

↓

Name

↓

National ID

↓

Confidence
```

ثم إنتاج:

```
CardData
```

---

# 8- AI Pipeline

الملف موجود.

ولم يتم ربط جميع المراحل بعد.

الـ Pipeline المخطط له:

```
Photo

↓

Detect Card

↓

Crop

↓

Preprocess

↓

Extract Regions

↓

OCR

↓

Extract Name & ID

↓

Match with Excel

↓

Return CardData
```
